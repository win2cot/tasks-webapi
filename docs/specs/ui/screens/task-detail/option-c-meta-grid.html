フルページ表示(直URL/タブ復元 fallback)
Tenant Admin でも業務タスクへの特権はありません(ADR-0005)。本画面の操作可否は 所有者 / 担当者 / 関係者 の3役割評価のみで決まります。

設計判断ポイント — 案C メタデータグリッド型(Meta Grid / Notion 風)

強み
全メタ項目をラベル+値のタイルで一画面に敷き詰め、情報密度が3案中最大。各タイルにホバーで ✏ が出て認可可能な項目だけ即編集に入れる。Notion / Asana 経験者に直感的で、項目の俯瞰性が高い。
弱み
密度が高いぶん視覚的な強弱が付きにくく、「次に何をすべきか」の誘導は弱い。狭幅では2→1カラムに畳まれ密度の利点が薄れ、ドロワー(狭幅)とフルページで体験差が出やすい。
想定読者
項目を素早くスキャン・一括編集したいパワーユーザー、PM。多数のタスクを高速にさばく運用フェーズ向き。
