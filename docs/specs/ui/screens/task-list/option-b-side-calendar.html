タスク一覧

URL は /tasks ・ 期限切れ常時表示 + 選択日リスト
Tenant Admin に切り替えてもサイドナビの運営メニューが増えるだけで、業務タスクの操作権限は変わりません(ADR-0005:所有者 / 担当者 / 関係者の3役割で評価。Tenant Admin に特権なし)。
フィルタ
OpenAPI GET /api/tasks parameters と一致
バッジ = その日期限の未完了件数 赤バッジ = 期限超過を含む日

設計判断ポイント — 案B 左サイドカレンダー(Side Calendar)

強み
月単位のカレンダーで件数バッジを一望でき、遠い未来日や月をまたぐジャンプもクリック一発。「いつどれだけのタスクがあるか」の俯瞰性が最も高く、上部フィルタバーは常時展開で絞り込み手段が常に見える。
弱み
左カラムに幅を取るためタブレット縦や狭い画面ではカレンダーが下に回り、一覧との同時視認が崩れる。画面要素が多く、S-03 ダッシュボードと並べると情報量がやや多め。
想定読者
期限を週・月単位で見渡したいリーダー・計画者層。先の予定を含めて日を行き来する頻度が高い PC 中心のユーザ。
新規タスク
URL は /tasks/new に同期(S-06)・初期値: 期限 = 表示中の日付
